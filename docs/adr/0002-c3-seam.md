# C3 seam: TypeScript engine, thin Rust shell

The TypeScript engine (ported from skill-pulse) owns all logic — Manifest read,
GitHub poll, Behind compute — and Rust owns only the genuinely-native slice: the
tray icon plus Behind badge, a background poll-clock (a tokio interval emitting an
event the webview listens for), and Keychain access for the GitHub token. The
seam is drawn so that **Rust never gates correctness**: every classification
decision happens in TS, where the engine already exists with its full test suite
and where the user — fluent in TS, new to Rust — can actually review it.

Status: accepted.

## Considered options

- **Port the engine logic into Rust.** Rejected: it re-derives a tested, working
  engine in a language the user can't yet review, putting correctness behind the
  weaker quality gate for no gain — the webview runs TS already, so the engine
  runs there for free.
- **Logic in Rust, TS as a thin view.** Rejected for the same reason, inverted:
  it maximizes the Rust surface that gates behavior.
